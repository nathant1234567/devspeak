## Commit Message Format
`<type>(<scope>): <subject>`

## Allowed Types
* **feat**: A new feature
* **fix**: A bug fix
* **docs**: Documentation only 
* **style**: Changes that do not affect funtionality (white-space, formatting, etc.)
* **refactor**: A code change that neither fixes a bug nor adds a feature
* **test**: Adding missing tests or correcting existing tests
* **chore**: Changes to the build process or adding tools and libraries
